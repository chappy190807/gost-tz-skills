# Чек-лист полноты ТЗ

Прогони готовый документ по всем пунктам. Для каждого проставь статус: **ОК** / **замечание** / **отсутствует**.

## Структура (для ГОСТ 34.602-2020)
- [ ] 1. Общие сведения (наименование, реквизиты, основания, сроки, финансирование).
- [ ] 2. Цели и назначение создания (развития) АС (цели измеримы).
- [ ] 3. Характеристики объектов автоматизации.
- [ ] 4.1. Требования к структуре АС в целом (подсистемы, режимы, диагностирование).
- [ ] 4.2. Требования к функциям (по каждой — содержание, регламент, критерий).
- [ ] 4.3. Требования к видам обеспечения (все виды рассмотрены или явно исключены).
- [ ] 4.4. Общие технические требования к АС (надёжность, безопасность, эргономика, защита от НСД и др.).
- [ ] 5. Состав и содержание работ (стадии, сроки, результаты).
- [ ] 6. Порядок разработки АС.
- [ ] 7. Порядок контроля и приёмки АС.
- [ ] 8. Требования к подготовке объекта автоматизации к вводу.
- [ ] 9. Требования к документированию.
- [ ] 10. Источники разработки.

## Структура (для ГОСТ 19.201)
- [ ] 1. Введение · 2. Основания · 3. Назначение.
- [ ] 4. Требования к программе (4.1–4.7 и специальные — рассмотрены или исключены с причиной).
- [ ] 5. Требования к программной документации.
- [ ] 6. Технико-экономические показатели.
- [ ] 7. Стадии и этапы разработки.
- [ ] 8. Порядок контроля и приёмки.

## Полнота содержания
- [ ] Нет разделов-заглушек с одной формальной фразой.
- [ ] Все метки `[ТРЕБУЕТСЯ УТОЧНЕНИЕ: …]` либо закрыты, либо явно перечислены как открытые вопросы.
- [ ] Цели создания сформулированы измеримо (число, срок, процент).

## Качество требований
- [ ] Каждое требование **проверяемо** (можно поставить «выполнено/не выполнено»).
- [ ] Нет размытых слов без критериев: «быстро», «удобно», «современно», «надёжно».
- [ ] Требования **атомарны** (одно требование — одна мысль).
- [ ] Требования **непротиворечивы** (нет конфликтующих пунктов).
- [ ] У ключевых функций заданы критерии приёмки.

## Оформление и стиль (ГОСТ 2.105)
- [ ] Нумерация разделов/подразделов — арабскими цифрами с точками.
- [ ] Изложение — безличная форма, настоящее время, «система должна…».
- [ ] Единая терминология (см. `reference/terminology.md`); один объект — один термин.
- [ ] Таблицы, перечисления, приложения оформлены по правилам.
