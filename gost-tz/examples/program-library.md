# Пример: ключевые разделы ТЗ на программу (ГОСТ 19.201-78)

Образец стиля и уровня детализации. Программа — утилита пакетной конвертации и валидации XML-документов (командная строка + библиотека).

## 3. Назначение разработки

**3.1. Функциональное назначение.** Программа предназначена для пакетной проверки XML-документов на соответствие XSD-схеме и их преобразования в формат JSON по заданным правилам сопоставления.

**3.2. Эксплуатационное назначение.** Программа используется инженерами интеграции в составе конвейеров обработки данных на серверах предприятия; запускается вручную оператором или планировщиком заданий.

## 4.1. Требования к функциональным характеристикам (фрагмент)

- **Состав функций.** Валидация документа по XSD; преобразование XML→JSON; пакетная обработка каталога; формирование протокола обработки.
- **Организация входных данных.** На входе — каталог с файлами `*.xml` в кодировке UTF-8 и файл XSD-схемы; максимальный размер одного документа — до 50 МБ.
- **Организация выходных данных.** На выходе — файлы `*.json` в указанном каталоге и текстовый протокол с перечнем обработанных, отклонённых и ошибочных файлов.
- **Временны́е характеристики.** Обработка документа размером до 5 МБ — не более 3 секунд на указанной конфигурации; пакет из 1000 документов — не более 15 минут.

## 4.2. Требования к надёжности (фрагмент)

- Программа контролирует корректность входных данных: файл, не соответствующий XSD, отклоняется с записью причины в протокол, обработка пакета при этом не прерывается.
- При аварийном завершении обеспечивается сохранность ранее обработанных результатов; повторный запуск не обрабатывает уже завершённые файлы.
- Устойчивость к некорректным входным данным: недопустимая кодировка или повреждённый XML не приводят к аварийному останову.

## 4.4. Требования к составу и параметрам технических средств (фрагмент)

- Процессор x86-64 не ниже 2 ядер; оперативная память — не менее 2 ГБ; свободное дисковое пространство — не менее двукратного объёма обрабатываемого пакета.

## 4.5. Требования к информационной и программной совместимости (фрагмент)

- Программа функционирует под управлением ОС Linux (ядро 5.x и выше) и Windows 10/11.
- Среда исполнения — [указывается: например, Java 17 LTS]; используемые библиотеки должны иметь лицензии, допускающие коммерческое использование (лицензионная чистота).
- Входной и выходной форматы: XML (по XSD 1.1), JSON (RFC 8259).
