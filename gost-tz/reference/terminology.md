# Глоссарий терминов ГОСТ

Единый словарь терминов для составления и проверки ТЗ. Один термин — одно значение по всему документу. Источники: ГОСТ 34.003-90, ГОСТ 19.781-90, ГОСТ 34.601-90, ГОСТ 34.602-2020, ГОСТ 34.603-92.

## Основные термины

- **Автоматизированная система (АС)** — система, состоящая из персонала и комплекса средств автоматизации его деятельности, реализующая информационную технологию выполнения установленных функций.
- **Объект автоматизации** — совокупность взаимосвязанных процессов, оборудования, подразделений и персонала, деятельность которых автоматизируется.
- **Функция АС** — совокупность действий системы, направленная на достижение определённой цели (обработка данных, формирование документа, контроль и т. п.).
- **Задача (АС)** — функция или часть функции АС, представляющая собой формализованную совокупность автоматических действий, выполнение которых приводит к результату заданного вида (ГОСТ 34.003-90, п. 1.4).
- **Подсистема** — часть системы, выделенная по функциональному или иному признаку.
- **Вид обеспечения АС** — совокупность средств и решений определённого типа. По ГОСТ 34.003-90 (раздел 2) виды обеспечения: организационное, методическое, техническое, математическое, программное, информационное, лингвистическое, правовое, эргономическое. (Метрологического обеспечения как отдельного вида в ГОСТ 34.003-90 нет; требования к нему могут задаваться в подразделе 4.3 ТЗ по ГОСТ 34.602, если АС выполняет измерения.)
- **Показатель назначения** — характеристика системы, отражающая степень соответствия её основному назначению и приспособленность к изменению условий применения.

## Требования и качество

- **Требование** — задокументированное условие или возможность, которой должна удовлетворять система (продукт). Должно быть проверяемым, атомарным, непротиворечивым.
- **Функциональное требование** — что система должна делать (действие, функция).
- **Нефункциональное требование** — каким свойством система должна обладать (производительность, надёжность, безопасность, эргономика).
- **Ограничение** — условие, сужающее пространство решений (бюджет, срок, технологический стек, нормативные акты).
- **Критерий приёмки** — измеримое условие, при выполнении которого требование считается реализованным.
- **Вариант использования (use case)** — описание последовательности взаимодействий действующего лица (актора) с системой для достижения значимой цели.
- **Действующее лицо (актор)** — роль пользователя или внешней системы, взаимодействующей с системой.

## Стадии, работы, приёмка

- **Стадия создания АС** — часть процесса создания системы, установленная нормативными документами и заканчивающаяся заданным результатом (по ГОСТ 34.601: формирование требований, разработка концепции, ТЗ, эскизный проект, технический проект, рабочая документация, ввод в действие, сопровождение).
- **Этап** — часть стадии, выделенная по соображениям единства характера работ и (или) завершающего результата.
- **Испытания** — экспериментальное определение количественных и качественных характеристик системы.
- **Предварительные испытания** — испытания для определения работоспособности системы и решения вопроса о возможности её приёмки в опытную эксплуатацию (ГОСТ 34.603-92, п. 1.8).
- **Опытная эксплуатация** — эксплуатация системы для определения фактических значений её характеристик, готовности персонала и готовности системы к приёмочным испытаниям (ГОСТ 34.603-92, п. 1.10).
- **Приёмочные испытания** — испытания для определения соответствия системы техническому заданию, оценки качества опытной эксплуатации и решения вопроса о возможности приёмки АС в постоянную эксплуатацию (ГОСТ 34.603-92, п. 1.11).

## Программа и программная документация (ЕСПД)

- **Программа** — данные, предназначенные для управления конкретными компонентами системы обработки информации в целях реализации определённого алгоритма (ГОСТ 19.781-90).
- **Программное изделие** — программа на носителе данных, являющаяся продуктом промышленного производства. Термин применяется в ГОСТ 19.201-78 («…к программе или программному изделию»); его определение восходит к отменённому ГОСТ 19.004-80 и в действующем ГОСТ 19.781-90 отсутствует. При строгой терминологии предпочтительны «программа» и «программное обеспечение».
- **Программный документ** — документ, содержащий сведения, необходимые для разработки, изготовления, сопровождения и эксплуатации программы.

## Запрещённые замены (используй левый термин)

| Правильно (ГОСТ) | Не использовать |
|---|---|
| функция | фича, фиче, feature |
| вариант использования | юзкейс, use-case (как жаргон) |
| пользователь / действующее лицо | юзер |
| требование | требуха, хотелка |
| производительность / быстродействие | перформанс |
| надёжность | стабильность (как синоним) |
| интерфейс пользователя | UI (без расшифровки при первом упоминании) |
