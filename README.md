# Навыки «ТЗ по ГОСТ» для Claude Code

Набор из пяти [Claude Code Skills](https://docs.claude.com/en/docs/claude-code/skills) для **составления и проверки технических заданий (ТЗ) по ГОСТ** — стандартам РФ на автоматизированные системы и программы.

Навыки автоматически подхватываются Claude Code, когда в диалоге появляются соответствующие триггеры (например, «составь ТЗ по ГОСТ» или «проверь это ТЗ на соответствие ГОСТ»).

## Состав набора

| Навык | Назначение | Когда срабатывает |
|-------|-----------|-------------------|
| **gost-tz** | Составление ТЗ с нуля: интервью → структура → наполнение → оформление → самопроверка | «составь / напиши / разработай / оформи ТЗ по ГОСТ» |
| **gost-validator** | Проверка готового ТЗ на соответствие ГОСТ, поиск пропусков, противоречий, непроверяемых требований | «проверь / оцени ТЗ», «что не так в ТЗ» |
| **requirements-engineer** | Сбор и классификация требований (функциональные / нефункциональные / ограничения), реестр требований | «собери требования», «классифицируй требования» |
| **system-analyst** | Проектирование архитектуры: use case, ER-модель, API, декомпозиция на модули | «спроектируй архитектуру», «построй use case / ER / API» |
| **document-editor** | Финальная редактура официального документа: единый стиль, безличная форма, унификация терминов | «отредактируй / вычитай документ», «приведи к единому стилю» |

**Минимальный рабочий набор — `gost-tz` + `gost-validator`.** Остальные три навыка опциональны: они работают и самостоятельно, и как фазы внутри `gost-tz`.

## Поддерживаемые стандарты

- **ГОСТ 34.602-2020** — ТЗ на создание автоматизированной системы (АС).
- **ГОСТ 19.201-78 (ЕСПД)** — ТЗ на программу (программное изделие).
- **ГОСТ 2.105** — правила оформления текстовых документов.
- **ГОСТ 34.003 / 19.781 / 34.601** — терминология и стадии создания АС.

## Установка

### Вариант 1. Вручную (в конкретный проект)
Скопируйте нужные папки навыков в `.claude/skills/` вашего проекта:

```bash
git clone https://github.com/chappy190807/gost-tz-skills.git
mkdir -p your-project/.claude/skills
cp -r gost-tz-skills/gost-tz gost-tz-skills/gost-validator your-project/.claude/skills/
# при необходимости — и остальные три навыка
```

### Вариант 2. Глобально (для всех проектов)
Скопируйте папки навыков в `~/.claude/skills/`.

После установки перезапустите Claude Code — навыки появятся в списке доступных.

## Структура репозитория

```
gost-tz/                 # главный навык — создание ТЗ
├── SKILL.md
├── reference/           # структуры ГОСТ 34.602, 19.201, 2.105, глоссарий
├── templates/           # пустые каркасы ТЗ на АС и на программу
├── checklists/          # вопросы интервью + чек-лист полноты
└── examples/            # примеры заполненных разделов
gost-validator/          # проверка готового ТЗ
├── SKILL.md
└── reference/           # правила проверки соответствия ГОСТ
requirements-engineer/   # сбор требований (опционально)
system-analyst/          # архитектура (опционально)
document-editor/         # финальная редактура (опционально)
```

## Как это работает

Навык `gost-tz` ведёт составление ТЗ по фазам: **интервью → развёртывание структуры нужного ГОСТ → наполнение разделов → оформление по ГОСТ 2.105 → самопроверка по чек-листу**. Тяжёлая нормативка (полные структуры стандартов, шаблоны, чек-листы, примеры) вынесена в отдельные файлы и подгружается только по мере необходимости (progressive disclosure), поэтому сам `SKILL.md` остаётся коротким.

Триггеры навыков не пересекаются: у каждого в поле `description` явно указано, **когда применять** и **когда передать работу другому навыку** (создание ↔ проверка ↔ сбор требований ↔ архитектура ↔ редактура).
